## 2024-06-18 - Improve Discoverability of Icon-Only Buttons
**Learning:** For icon-only standard Android views (like MaterialButton, outside of MenuItems), the `app:tooltipText` attribute is often insufficient or unsupported, especially on older API levels. Relying solely on `android:contentDescription` meets basic screen reader accessibility but leaves non-screen reader users without visual cues on long-press or mouse hover.
**Action:** When adding tooltips to non-menu icon-only standard views in Android standard layouts, consistently mirror the `android:contentDescription` using the standard `android:tooltipText` attribute (available since API 26) to provide robust, visually accessible tooltips without relying on custom libraries or potentially ignored namespace attributes.

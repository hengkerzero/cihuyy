## 2025-05-10 - Add floating label hint to add favorite dialog
**Learning:** Found an `EditText` in the `dialog.xml` used for adding favorites that lacked any hint or input type, which is bad for screen readers and general usability. Material Design `TextInputLayout` is available and provides a much better experience with floating labels. Also learned that to pass `lint` check locally for Android U+ broadcast receivers, the receiver registration needed to be updated to explicitly include `ContextCompat.RECEIVER_EXPORTED` flags, though this is primarily an OS compatibility requirement.
**Action:** Always check form inputs for proper hints (`android:hint`), input types (`android:inputType`), and consider wrapping them in `TextInputLayout` when appropriate. When linting flags issues unrelated to UX, fixing them correctly helps to ensure a clean build pipeline.
